# Security Policy

| Version | Supported |
| ------- | --------- |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report

Please create a private security report through GitHub Security Advisories:

- Go to the repository Security tab
- Click "Report a vulnerability"
- Provide detailed reproduction steps and impact information

Alternatively, you may open a GitHub Issue for non-sensitive security concerns.

---

## Response Process

Security reports will be reviewed as soon as possible.

Typical process:

1. Vulnerability triage
2. Risk assessment
3. Fix development
4. Validation and testing
5. Public disclosure (if applicable)

---

## Scope

This repository is a learning and demonstration project for DevSecOps practices including:

- SAST (Bandit)
- DAST (OWASP ZAP)
- GitHub Code Scanning
- SARIF reporting
- CI/CD security integration
